# Select

# Default

Documentation Vuesax Alpha

These docs refer to the Vue 3 support, to see the documents of the previous versions you can do it here 👉 Vuesax (4.0+)

Add a select element with the vs-select component and the vs-option sub component

Select
Code Copied
  • Template
  • Script
  • Style
  • All 
<template>
  <div class="center con-selects">
    <vs-select v-model="value" placeholder="Select">
      <vs-option label="Vuesax" value="1"> Vuesax </vs-option>
      <vs-option label="Vue" value="2"> Vue </vs-option>
      <vs-option label="Javascript" value="3"> Javascript </vs-option>
      <vs-option disabled label="Sass" value="4"> Sass </vs-option>
      <vs-option label="Typescript" value="5"> Typescript </vs-option>
      <vs-option label="Webpack" value="6"> Webpack </vs-option>
      <vs-option label="Nodejs" value="7"> Nodejs </vs-option>
    </vs-select>
  </div>
</template>

# State

Change the style of the component to the color passed in the state property, the allowed colors are only the main ones of vuesax

This property can be used to indicate a missing field to the user or when something is ready.

Select
Select
Select
Select
Select
Code Copied
  • Template
  • Script
  • Style
  • All 
<template>
  <div class="center con-selects">
    <vs-select
      v-for="(color, index) in colors"
      :key="index"
      v-model="color.value"
      :state="color.color"
      :label="color.color"
      placeholder="Select"
    >
      <vs-option label="Vuesax" value="1"> Vuesax </vs-option>
      <vs-option label="Vue" value="2"> Vue </vs-option>

# API

PROPS

PropertyTypeValuesDescriptiondefaultExample More
colorColorMain colors of vuesax RGB HEXChange the color of the component.primary Usage
loadingBooleantrue falseDetermine if the component is in the loading state and add an animation.false Usage
flipNewBooleantrue falseChanges the placement of the options element to keep it in view.true Usage
teleportedNewBooleantrue falsewhether select dropdown is teleported to the bodyfalse
placeholderStringStringAdd a placeholder to the component. Usage
labelStringStringAdd a label to the composite select. Usage
label-placeholderStringStringAdd a placeholder that when in focus or with value becomes a label. Usage
filterBooleantrue falseAdd the functionality to filter the select options.false Usage
default-first-optionNewBooleantrue falseselect first matching option on enter key. Use with `filter`false Usage
allow-createNewBooleantrue falseWhether creating new items is allowed. To use this, `filter` must be truefalse Usage
multipleBooleantrue falseAdd the functionality of being able to select several options from a select.false Usage
multiple-limitNewNumbernumberMaximum number of options user can select when `multiple` is `true`. No limit when set to 00 Usage
stateStringVuesax main colorsChange the state of the component to the color provided.false Usage
disabledBooleantrue falseDetermine if the component is in the disabled state.false
collapse-chipsBooleantrue falseWhether to collapse tags to a text when multiple selectingfalse Usage
max-collapse-chipsNewNumbernumberThe max tags number to be shown. To use this, collapse-chips must be `true`1 Usage
option-group:labelNewStringStringSet label for select group (required) Usage
